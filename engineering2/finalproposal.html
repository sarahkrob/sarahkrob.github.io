Final Project Proposal

For my final project, I will be recreating Asteroids. I feel like the movement system I devised suits that sort of movement well. There will be a spaceship object that is controlled by the player, and that can shoot bullet objects to hit asteroids that fly across the screen randomly. There will be no score, and probably no win or lose condition.

I will use my system to do the majority of this game, as I can expand on it to make it work for most aspects of it. The movement is already suited for the game, so I just need to add the ability for objects to move without player input, which shouldn’t be too difficult. Beyond that, I just need to be able to spawn objects off the screen and when the player hits a key. 

The system I would like to use is Saransh Wali’s audio system (https://www.googleapis.com/drive/v3/files/1Ri4VNFgaFxRzLjn83LB93_q1NfcqHFoa?alt=media&key=AIzaSyCJCGsVYlXbeoHltquXAIlqbBSCpwGXuVg), to play a sound when shooting bullets on a key press and a sound when asteroids are destroyed by a bullet. I would ideally like to use Josh Butner’s collision system as well (https://www.dropbox.com/s/5s3ar7f4elxb1yv/EAE%206320%20Engine%20System%20Final%20Writeup.pdf?dl=0), but because my project basically replaces the rigidbody system that we were provided that he uses, I’m not sure if that’s viable. If not, I’ll probably just quickly add rudimentary 2D AABB collisions into my own system.
