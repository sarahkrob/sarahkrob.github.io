For this assignment, I moved the remains of the platform specific code in Graphics.cpp into its own class, and was able to consolidate the rest of it into one single Graphics.cpp file. To do this  I actually just made changes that I mentioned wanting to make in my last writeup, so everything worked out great. I made a class called cRenderer that operates on the buffers (clearing and swapping them) and also initializes the viewport for Direct3D, which is done in OpenGL’s effect initializer instead. That was the one remaining large method that needed to be taken out. Besides that, I made a ClearBuffer method that takes a color to clear with as a parameter, and a SwapBuffer method. Renderer is a separate class with its own separate .cpp implementation files per platform, as there were enough differences and the methods were short enough that I felt it was warranted to break it up. For OpenGL, which didn’t have an initialize viewport method that needed to be moved, the initialization for the renderer object simply does nothing.


I did leave a small amount of platform-specific code in Graphics.cpp with if statements to swap it, mostly because it was short enough I didn’t feel it warranted entire new methods for each platform. 


In order to initialize an effect, the user only needs to provide the path to the effect. For a mesh, the user needs to provide the vertex data and index data, both in left-handed order, as well as the number of triangles. I considered passing the number of vertices as well, but ultimately for now I know the number of vertices per triangle is always going to be three so I can perform that calculation in my initialization and drawing code for the mesh. In the future I may want to change that to be passed as well so no numbers are hardcoded, but I don’t actually know if there’s ever a case where there will be more than 3 vertices per triangle. 


An effect takes up 16 bytes on OpenGL and 48 bytes on Direct3D. A mesh takes up 4 bytes on OpenGL and 8 bytes on Direct3D.

I believe the reason effects cost so much more on Direct3D is that the cShader object has its own platform-specific variables, which on OpenGL is one int but on Direct3D is a union of two structs. The cRenderState object also only has specific data for Direct3D, which I believe is increasing the size. As for meshes, Direct3D contains a few specific structs whereas OpenGL just has three unsigned ints and a short.
