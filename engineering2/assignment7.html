In this assignment we had to change our project to be able to import meshes from Maya and format them to match the mesh file format we set up last week. We did this by writing the code that formatted the data coming in from Maya, and creating a plugin that can be used and debugged with Maya.


I did not have to add any references to my MayaMeshExporter project, nor did any of my existing projects depend on MayaMeshExporter. I’m not sure if this was the intended solution, but my mesh exporter does work correctly to get the information needed to make meshes, so who knows. I can’t see any reason it would need information from elsewhere in my code, or vice versa. 


In regards to the unused data from Maya, I chose not to import it for now. I could add it in later if it was ever needed for rendering (I would like to add color in, for example, but didn’t have time to figure that out after debugging the base project, so I’ll be adding it later) but I wanted to keep my human-readable mesh file as simple as possible. Adding a bunch of extraneous information when all that my program currently uses is the vertices and indices seemed pointless to me and would just make the file more confusing for someone to read. 


When you try to load a model with too many vertices in my program, it throws an assert. I’m not sure if this is the intended “graceful handling” but I didn’t want to be running a program that was using too much memory or something like that, so I thought an assert was safe for now. Another solution could have been to just backpedal any of the rendering happening on a mesh that hit the vertex limit, but I think that might confuse users if they didn’t know why their mesh wasn’t rendering. 


As a note, my project still only runs on x64 Debug. There are also a few issues regarding when one tries to move the moveable object in my scene (the sphere), but nowhere in the requirements does it mention being able to move one object so I feel confident turning it in like this. I am currently trying to debug the movement issue.
