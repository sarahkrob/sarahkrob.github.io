Engine System Update 2


For this update, I’ve implemented acceleration into my physics system. This means linear movement is now affected by friction, and rotational movement is now affected by a nebulous angular drag number. These are all hard-coded for now, which is something for me to change in the upcoming week. I’ve also implemented torque, with the ability for the user to apply a force at a point in the scene and have that affect both the position and rotation of the object. This is what gave me the most trouble, as getting my torque to behave correctly and not make the object rotate infinitely was difficult. I’m still not satisfied with my solution to the problem, as it seems a little hacky, but it’s temporarily fixed my issue. I’d like to look at this further in the next week, and also find a way, if possible, to allow users to click where they’d like a force to be applied from, but I have no idea how difficult this might be, as I’ve not worked with clicking. Another change is to update my objects’ moment of inertia value, currently hard-coded, to use values sent in from the mesh of the object to calculate the size of the object. Luckily, no new information should need to be sent for this. There’s also some stuff with use of tangents and normals to apply friction that I’d like to look into further (I have been extensively in the last few days, but it’s very new to me and taking a lot of brainpower to comprehend) but I have no idea if it’s worth it given that I’m generally satisfied with my current movement, assuming the user will be able to customize it. I also need to test further with different meshes, once my moment of inertia implementation is changed. Basically, I have a lot to do still!
